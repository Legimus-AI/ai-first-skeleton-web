# Feature Plans

Every non-trivial feature starts with a plan before any code is written.

## Workflow

1. Copy `TEMPLATE.md` → `<feature-name>.md`
2. Fill in objective, context, approach
3. Declare confidence (1–10). If < 5, stop — plan only, no implementation
4. Get human approval
5. Implement, checking off verification checkpoints
6. Delete the plan file after merge (plans are ephemeral, not documentation)

## When to Plan

- New slice or feature
- Architectural changes
- Changes affecting multiple files
- Confidence < 7 on approach

## When NOT to Plan

- Bug fixes with obvious cause
- Single-file changes
- Documentation updates
